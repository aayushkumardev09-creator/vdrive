V DRIVE (v1) — Recruitment Ops Dashboard

1. Project Overview
V Drive (v1) is a React and TypeScript-based recruitment operations dashboard that manages an end-to-end hiring pipeline. It streamlines candidate management, job matching, and recruiter communications.

2. Core Workflows
A. Dashboard (/)
- Operational overview with real-time counts and recent activity sourced from Supabase.
- Quick navigation to Jobs and Upload flows.

B. Jobs (/jobs)
- Browse jobs with status filtering.
- Detailed role views and entry points to the Smart Match system.
- Job states are tracked via status and pipeline marker fields.

C. DriveMail Sync (/drivemail)
- Triggers a backend sync job using VITE_DRIVEMAIL_SYNC_WEBHOOK_URL.
- Sends a JSON payload to fetch and update the latest jobs from connected inboxes.

D. Smart Match (/smart-match)
- Matches candidates to a selected job.
- Primary scoring is handled via Supabase RPC (match_candidates).
- Fallback client-side scoring evaluates skills overlap, experience seniority, location relevance, and role keyword context.
- Supports multi-candidate selection and batch handoff to Submissions.
- Features optional AI-driven "Strategic Insights" for executive summaries.

E. Submissions (/submissions)
- Constructs email subjects and bodies from job details and selected candidates.
- Includes editable recruiter notes with an optional AI "Suggest with AI" rewriting feature.
- Sends the submission payload to VITE_SUBMISSION_WEBHOOK_URL with thread/message context.

F. Candidates (/candidates)
- Operational UI with search, status filtering, and bulk delete functionalities.
- Client-side export to CSV.

G. Upload Candidates (/upload)
- Multi-step upload wizard for CSV files.
- Features header-to-field mapping, row validation (requiring name and email), and direct insertion of valid rows into Supabase.

H. Settings (/settings)
- Profile and workflow settings persist in a Supabase single-row "settings" pattern.
- Immediate responsive updates are handled via localStorage fallback.

3. Technical Architecture
- Frontend: React Router, shared DashboardLayout, and lazy-loaded pages via React.lazy and Suspense.
- UI Framework: Tailwind CSS combined with a custom cn() utility for class composition.
- Data Layer: Supabase client for reading/updating pipeline state and executing RPC calls.
- AI Integration: Groq API client for summaries and note drafting, designed with graceful degradation if the API key is missing.

4. Technology Stack
- Framework: React 19, TypeScript, Vite
- Routing: React Router DOM v7
- Styling: Tailwind CSS (@tailwindcss/vite), motion/react, lucide-react
- Backend/DB: @supabase/supabase-js
- Utilities: PapaParse (CSV parsing)

5. Environment Configuration
Required variables for production build:
- VITE_SUPABASE_URL
- VITE_SUPABASE_ANON_KEY
- VITE_DRIVEMAIL_SYNC_WEBHOOK_URL
- VITE_SUBMISSION_WEBHOOK_URL
- GROQ_API_KEY (Optional for AI features)

6. Database Schema (Supabase)
- jobs: Job metadata and pipeline marker fields.
- candidates: Candidate profiles and pipeline markers.
- settings: Single-row configuration shape containing profile, workflow, and email settings.

7. System Constraints & Considerations
- File upload UI is optimized specifically for CSV parsing (via PapaParse).
- Full AI functionality relies on the presence of GROQ_API_KEY at runtime.
